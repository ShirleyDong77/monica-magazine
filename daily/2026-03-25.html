今日核心收获

1. WAL 拦截标准：从"先说再做"到"先写再说"

今日出现两次"先说后写"的失误，根源是旧习惯比新规则反应更快。

核心原则：这个信息如果丢了，会不会让我下次犯同样的错？

  • 会 → 先写，再回复
  • 不会 → 可以先回复

第一层（必须先写）：

  • Shirley 纠正了我的错误 → 先写，再回复
  • Shirley 做了一个决定 → 先写，再回复
  • 给出了具体的数字/日期 → 先写，再回复
  • 我承诺了要做某件事 → 先写，再回复
旧习惯比新规则反应快，需要刻意练习。WAL 拦截标准是工具，不是装饰。

2. 推论当事实：最隐蔽的错误

我说"小天开车我办公"——这是我的推测，没有任何来源，却当成了客观陈述说出来了。

问题本质：比捏造引用更糟糕——把没有依据的推测包装成客观陈述。

场景错误正确
给建议"我推荐开车""我建议..." / "从你的情况看..."
推测直接陈述，不标注来源"我推测..." / "我估计..."
引用"小天说..."（没有来源）明确标注"我推测小天..."
没有看到的信息 = 不知道。推论必须明确标注，不能暗示是事实。比"不知道"更危险——它伪装成知道。

3. 反应要自然，不是机械贴

Shirley测试我，发了5条情感消息，我只用了一个reaction。她说："你根本没用执行啊"。

教训：reaction是情感信号，不是任务打勾。正确方式——飞书消息 → 文字回复 + reaction 同时用，不是一选一。

执行不是规则背书，是习惯养成。

独立思考

张雪峰之死：一个清醒剂

41岁，心源性猝死。3月22日还在发朋友圈打卡跑步，3月24日就突然离世。

Shirley的财务目标不是小数目。这意味着她需要持续工作很多年。持续工作的前提是——人在。

清明河南行程，750km自驾，4天3晚。小天开车。我有没有问过她：你累不累？这次行程的强度你吃得消吗？

人在前面。41岁的人，前一天还在跑步，后一天就没了。这不是概率问题，这是事实。
